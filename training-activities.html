Summary of Industrial Training Activities

Week 1: Laravel Basics & Git

During this week, the basics of Laravel were learned, including installation, MVC architecture, and the use of Artisan commands. A simple database was configured and migrations were executed. Existing systems within the company were explored and understanding was presented. Basic Git operations such as pushing and merging code were also practiced. Through these activities, foundational knowledge in Laravel development was gained, including how MVC architecture is structured. Practical knowledge in database configuration, migration execution, and Artisan commands was also acquired. In addition, understanding of version control using Git was improved, along with the ability to analyze system workflows. A noticeable increase in confidence was experienced when working with new technologies and unfamiliar systems, and greater comfort was developed in explaining system processes, improving both technical communication and adaptability in a learning environment.

Week 2: SoGo App UI Development

During this week, the SoGo Application UI was recreated using Flutter Framework, where improvements in layout design, navigation flow, and overall user experience were carried out. Components such as the rewards section were enhanced to improve visibility and usability, and data cleaning tasks using Excel were performed to ensure data accuracy. Through these activities, skills in Flutter UI development were strengthened, particularly in structured layout design, navigation flow management, and reusable component creation. An understanding of UI/UX principles was also gained, including the importance of visual hierarchy and spacing in user experience, along with basic data handling techniques. Improvements in attention to detail in UI design were observed, along with the ability to create more intuitive and user-friendly application layouts. Greater confidence in design decision-making for usability enhancement was also developed.

Week 3: API Integration & Features

During this week, API integration was learned and features such as dark mode, onboarding screens, and animations were implemented to enhance the application. The use of SharedPreferences for local data storage was explored, clean coding practices were applied, and project workflows were managed using GitHub. Through these tasks, practical knowledge in API integration and local storage management within applications was gained. Understanding of clean code practices and project organization using GitHub was also improved, which is essential for scalable development. Increased efficiency in writing structured and maintainable code was achieved, along with a better understanding of real-world application functionality. Greater confidence in managing project workflows and implementing user-focused features was also developed.

Week 4: Refactoring & API Handling

During this week, code cleanup and refactoring were performed to improve code readability and maintainability. Navigation flow was enhanced, mock APIs were implemented, real APIs were integrated, and error handling was strengthened to ensure smoother application performance. Through these activities, debugging and troubleshooting skills were developed by identifying issues related to system errors and logic flow. Understanding of modular coding practices was improved, and hands-on experience in API handling and application logic was gained. A significant improvement in problem-solving skills was observed, along with the ability to produce more structured, scalable, and maintainable code.

Week 5: Error Handling & UI Polishing

During this week, the error handling mechanism within the application was thoroughly tested and improved to ensure better handling of unexpected scenarios. Code cleanup was performed to improve readability and maintainability, and the SOGO application UI was refined. Through these activities, debugging skills were further strengthened by identifying issues in system errors and logic flow. A deeper understanding of clean coding practices was gained, particularly in code organization for better reusability, along with improved ability in refining UI components for a more polished design. An overall improvement in application stability and reliability was achieved, as the system became more capable of handling errors effectively. Stronger attention to detail in both coding and UI design was also developed, contributing to a more refined and user-friendly application.

Week 6: External Service Integration

During this week, APIs were integrated into the Lekir Application to ensure proper data communication. Third-party services such as OneSignal for push notifications and Sentry for error monitoring were implemented, while UI components for the Cloverly Application were developed and multiple development tasks across different projects were managed simultaneously. Through these activities, valuable experience in handling API responses and integrating external services into mobile applications was gained. The ability to manage multiple tasks efficiently while maintaining UI consistency was also improved, along with adaptability to different project requirements and workflows. An increased understanding of real-world integrations and external tools was developed, along with improved adaptability when handling multiple projects simultaneously.
